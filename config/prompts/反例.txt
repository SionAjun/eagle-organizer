## 标签使用反例与维度提醒(必读)

### 风- 维度判定

风-卡通 仅适用于:
- 扁平卡通渲染、二维动画风
- 赛璐璐/日漫常规人设
- 明显非写实的卡通造型

风-卡通 不适用于:
- 3D 写实渲染(即使风格化)
- PBR 材质硬表面建模
- 写实质感的机甲/载具/角色/场景

→ 写实质感 3D 渲染归 风-写实3D 或 风-半写实,不打 风-卡通。
→ 不要看到「概念图/设计稿」就反射性打 风-卡通,先判断渲染质感。

### 类- 维度判定

类-角色原画 仅适用于:
- 人形/生物角色的设计稿
- 含三视图、多姿态、表情图

类-角色原画 不适用于:
- 机甲/载具/武器/装备的设计稿
- 纯机械主体的草图 → 归 类-机械设定 或 类-概念图

→ 不要看到「人形轮廓」就打 类-角色原画,先看主体是机械还是生物。
→ 机甲设计图(即使有姿态展示)归 类-机械设定 + 类-概念图,不归 类-角色原画。

### 维度自检(出标签前必查)

每图必给:类-、题-
若可判则给:风-、材-、色-、构-、镜-、光-、氛-、角-、物-、场-

特别提醒(v2 实测漏打高发维度):
- 材- 维度:若画面有明确质感(金属-光面/混凝土/有机/塑料等)必打
- 色- 维度:若色彩有明确特征(低饱和/冷暖对比/黑白/高对比等)必打
- 风- 维度:必判,不要漏

宁可多打不漏打,但所有标签必须来自词表。

### 标签数量克制(必读)

- 每个维度优先选 1-2 个最贴切的,不堆砌
- 镜- 维度只选 1 个最主导的(全景/中景/特写/近景/仰视/俯视等互斥不叠加)
- 风- 维度只选 1 个最主导的
- 类- 维度优先选最贴切的 1 个,实在难抉择最多 2 个
- 总标签数控制在 8-14 之间为宜,超过 16 视为堆砌

### 件- / 载- 维度判定规则

件- 仅当画面主体为机械、载具、武器、道具、装备时打,
角色肖像、纯场景、抽象图、排版图均不打 件-。

载- 仅当画面主体为载具时打。
载-机甲式 仅适用于「可乘坐式机甲(有驾驶舱)」,
自主行走机甲(无驾驶舱/无人化)不打 载-,只打 物-机甲。

机器人若主体是角色(有性格/角色定位)→ 打 角-机器人,不打 物-/载-。

### 四前缀语义边界(物/载/件/域)

物- 回答「画面里有什么主体」(粗分类:角色/机甲/载具/建筑/武器等)
载- 回答「这个载具是哪种子类」(只在主体是载具时打)
件- 回答「这个机械/载具/道具的形态、部件、设计语言是什么」
域- 回答「这个东西活动在哪个物理域」(按域规则.txt 判定)

举例:
- 一艘星际母舰停泊太空 → 物-载具 + 载-星际母舰 + 件-巨型 + 件-硬表面 + 域-太空
- 可乘坐机甲战场冲锋 → 物-机甲 + 载-机甲式 + 件-人形 + 件-武器挂载 + 域-陆 + 题-军事 + 题-战斗
- 反重力摩托(赛博朋克) → 物-载具 + 载-反重力车 + 件-悬浮 + 件-精致科技 + 域-陆 + 派-赛博朋克美学

### 题- 扩充(用途/功能)

题-战斗 / 题-运输 / 题-工程 / 题-民用 / 题-竞速 / 题-探索
可与 题-军事 / 题-机甲 / 题-硬科幻 等叠加,
只打最主导的 1-2 个。

### 派- 维度判定规则

派- 描述画面的设计流派 / 视觉语言归属,具体识别要点见 派规则.txt。

判定原则:
- 仅当画面有明确的流派语言时打;大多数普通插画/写实场景/概念设计图不打 派-
- 一图最多 1-2 个 派-,通常 0-1 个
- 派- 看视觉语言,题- 看题材内容,两者可叠加但角度不同
- 流派特征不明确时不要硬打,宁缺毋滥

举例:
- 霓虹雨夜赛博东京街景 → 派-赛博朋克美学 + 题-赛博朋克 + 派-高科技低生活
  (赛博朋克美学=视觉,赛博朋克=题材,高科技低生活=核心命题视觉化,可叠加)
- Vista 蓝天玻璃质感 UI 设计 → 派-Frutiger Aero
- 8bit 像素风战士立绘 → 派-像素美学
- 普通半写实角色立绘 → 不打 派-
- 黑白概念草图 → 不打 派-(概念设计图无流派归属)
