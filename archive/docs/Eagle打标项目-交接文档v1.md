## 6. 关键命令(cheatsheet)

```bash
# 主流程
python tag_real.py --prepare --limit 20    # 拉 20 张,生成 pending.json
python tag_real.py --apply-batch           # 读 batch_results_NNN.json 写回 Eagle
python tag_real.py --checkpoint            # 手动触发检查点

# 自动化清理(v2.1+)
python tag_real.py --cleanup --dry-run     # 预览
python tag_real.py --cleanup               # 实际执行

# 回溯流程(词表升级后用)
python tag_real.py --build-review-queue    # 生成回溯队列
python tag_real.py --review-prepare --limit 10
python tag_real.py --review-apply --batch 01
python tag_real.py --review-report
```

文件命名:`batch_results_NNN.json`(NNN = batch_counter,自增,3 位 padding)。脚本会自动归档已用过的批次到 `archive/batches/`。

## 7. 已踩过的坑(不要再踩)

| 坑 | 解决 |
|---|---|
| 系统代理劫持 localhost (127.0.0.1:7892) | `urllib.request.ProxyHandler({})` |
| Eagle API limit 小值静默截断 | 统一 limit=1000 |
| Eagle API 无 total 字段 | 大 limit (25000) 探底 |
| `orderBy=-CREATEDATE` 反向返回 1970 | 不传 orderBy |
| btime 字段 = 0 | 改用 modificationTime |
| `/api/v2/` 不存在(Eagle 4.0) | 全用 `/api/` |
